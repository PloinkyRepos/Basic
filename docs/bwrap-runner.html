Overview / bwrap-runner

bwrap-runner

bwrap-runner is a Basic catalog agent that exposes one MCP tool, sandbox_exec, for bounded command execution inside an inner bubblewrap sandbox. It is sandboxed job delegation, not a replacement for Ploinky's whole-agent lite-sandbox: true backend.

Runtime Shape

The manifest runs through normal Ploinky container execution. The image is resolved from BWRAP_RUNNER_IMAGE, whose default is ploinky/bwrap-runner:node20-bookworm. The default profile runs scripts/build-image.sh as a host preinstall hook so the local Podman or Docker image is built before container creation if it is not already present.

Sandbox Policy

The tool wrapper accepts only command, stdin, timeoutMs, and allowlisted env entries. It rejects extra fields such as raw mounts, binds, capabilities, network selectors, or direct bubblewrap flags. The generated bubblewrap argv is built by bwrap-runner/lib/policy.mjs so policy construction can be tested without invoking bubblewrap.

Each job receives a private state directory under BWRAP_RUNNER_STATE/jobs/<jobId>/. The inner sandbox gets a writable /work, writable /outputs, a tmpfs /tmp, /proc, /dev, and read-only binds for selected system paths that exist in the runner image. The wrapper starts the child process with an empty outer environment and lets bubblewrap set only validated environment keys.

Readiness And Limits

The manifest agent command runs bin/healthcheck.mjs before AgentServer.sh, and healthcheck.sh wires the same probe into Ploinky readiness checks. The probe runs a real nested-bwrap command. If the host kernel or outer OCI runtime blocks nested user namespaces, the agent fails startup or readiness with a structured error and guidance.

The image supplies bwrap, but it cannot change host kernel sysctls, Docker or Podman seccomp policy, AppArmor policy, or rootless runtime namespace rules.

Verification

The focused test suite validates argument policy, environment clearing, timeout limits, output tail retention, invalid payload errors visible to MCP callers, manifest startup gates, Basic catalog placement, and the nested-bwrap health probe. The health probe test skips when the local host does not provide /usr/bin/bwrap or nested namespace support.

The normative contract for this agent is DS002 bwrap Runner Agent.
