Overview

Basic Agent Catalog Documentation

The Basic repository is a Ploinky agent catalog. Each top-level directory is an agent package that can be cloned into a workspace under .ploinky/repos/basic/ and resolved through the normal Ploinky agent registry.

Repository Role

The catalog keeps agent implementations self-contained. A Basic agent owns its manifest, local scripts, optional MCP configuration, runtime support files, and agent-local documentation. Shared runtime behavior such as container startup, profile merging, host lifecycle hooks, routing, and invocation security remains in Ploinky itself. 

flowchart LR
    Workspace[Ploinky workspace] --> Repo[.ploinky/repos/basic]
    Repo --> Agent[Agent directory]
    Agent --> Manifest[manifest.json]
    Agent --> Runtime[Local scripts and MCP config]
    Manifest --> Ploinky[Ploinky runtime]

Agent Layout

Most existing agents are simple container or shell entries with a single manifest.json. Agents with larger behavior, such as bwrap-runner, keep policy modules, command wrappers, smoke checks, and documentation inside their own directory.

Path Purpose
<agent>/manifest.json Declares image, entry command, profile hooks, ports, mounts, resources, and readiness.
<agent>/mcp-config.json Declares MCP tools when an agent exposes tool commands through AgentServer.
tests/unit/ Contains focused Node tests for agent-local policy and smoke behavior.
docs/specs/ Contains DS specifications. The matrix is reachable through specsLoader.html?spec=matrix.md.

Coding And Tests

DS001-coding-style.md is the coding-style authority for this repository. JavaScript modules use ES module syntax, four-space indentation, and focused functions with narrow responsibilities. Tests should be placed under tests/unit/ when they validate reusable policy, manifest, or wrapper behavior without needing a full Ploinky workspace.

Run the bwrap-runner tests with node --test tests/unit/bwrapRunnerPolicy.test.mjs tests/unit/bwrapRunnerSmoke.test.mjs. The nested-bwrap smoke test skips cleanly when /usr/bin/bwrap or nested user namespaces are unavailable.

Specifications

The DS set is the source of truth for repository contracts. Start with DS000, then DS001, and use the specification matrix to reach agent-specific contracts.
